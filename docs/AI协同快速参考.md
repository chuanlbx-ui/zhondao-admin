# AI协同开发 - 快速参考卡

## 📋 5分钟速览

### 我的角色是什么？
```
🎨 前端开发AI → 实现UI和React组件
⚙️  后端开发AI → 实现API和业务逻辑
🧪 测试AI    → 编写和运行测试
🏛️  架构师AI  → 代码审查和设计决策
📖 文档AI    → 更新API文档和指南
🎯 协调AI    → 管理任务和解决冲突
```

### 我的任务是什么？
```
1. 打开 管理后台AI协同开发计划.md
2. 从列表中选择一个状态为"⏳ pending"的子任务
3. 更新任务状态为"🔄 in_progress"
4. 创建feature分支：git checkout -b feature/TASK-XXX-description
5. 按照AI协同开发实施指南执行
```

## 🚦 状态指示

### 任务状态
```
⏳ PENDING   → 等待开始
🔄 PROGRESS → 正在进行（我在做）
👀 REVIEW   → 代码审查中（在等待）
🟢 READY    → 准备合并
✅ DONE     → 已完成
🚫 BLOCKED  → 被阻塞（等待依赖）
```

### 代码审查状态
```
📝 REVIEW   → 等待审查
💭 COMMENT  → 有评论需要回复
✅ APPROVED → 已批准，可合并
❌ CHANGE   → 需要修改后重新审查
```

## 📞 遇到问题时

### 我的任务超期了
```
→ 通知 @协调AI
→ 描述：原因、剩余工时、建议方案
→ 等待协调和重新分配
```

### 我需要其他AI的帮助
```
→ 在GitHub issue或PR中@相关AI
→ 描述：具体需要、预期工时、deadline
→ 提供link和上下文
```

### 我发现了冲突
```
→ 立即通知 @协调AI @相关AI
→ 描述：冲突文件、原因、影响范围
→ 不要自己解决，等待协调
```

### 代码审查意见我不同意
```
→ 保持尊重但表达观点
→ "我理解你的意见，但我认为...因为..."
→ 最终由架构师AI决策
```

## 📊 工作时间表

### 日常流程
```
09:00 → 查看協調AI的日報和分配任務
10:00 → 開始開發（如果有待開始的任務）
12:00 → 中期檢查（需要時）
17:00 → 自測和同行代碼審查
18:00 → 提交代碼、更新進度、發送日報
```

### 代码审查周期
```
提交PR → 1-2小时 → 架构师AI审查
有反馈 → 30分钟 → 修改代码
重新审查 → 1小时 → 通过或再次反馈
通过 → 立即合并到main
```
